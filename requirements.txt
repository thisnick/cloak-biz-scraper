# The MCP server. Three pins, not one, and the two extra ones are load-bearing:
#
#   * mcp declares `starlette>=0.27` with no upper bound, so a fresh install
#     resolves the newest (1.3.x) — which fastapi 0.115.6 forbids (<0.42). The
#     result is not a resolver warning you can ignore: `import fastapi` itself
#     fails. Pinning starlette is what keeps the two compatible.
#   * sse-starlette 3.x requires starlette>=0.49.1 and would drag the same
#     conflict back in through mcp's dependency on it. 2.x sits with 0.41.
#
# `pip check` passes on this set. Loosen any one of them and it stops.
